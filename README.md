CS student at USTP in Cagayan de Oro, graduating 2027. I build full-stack apps, usually ones with a real-time part, since that's what I like figuring out most.

## Projects

**KABOOOM!** A real-time multiplayer word game with countdown rounds. Has a lobby system where players can create or join rooms and change settings. Built with Django Channels and WebSockets on the backend, React (TS) on the frontend, PostgreSQL for the database. The hardest part was keeping lobby state the same across all players without breaking when someone reconnects.

**Barangay Complaint System** A complaint management app for local government use. Uses Firestore for real-time data, has admin login, and a dashboard for barangay officials to manage complaints. I also set up a CI/CD pipeline with GitHub Actions that runs 15 unit tests, builds, and smoke tests, then deploys to Vercel with no manual steps.

**Steezy Gadget Hub** An e-commerce site with product filtering and infinite scroll. Users can sign in with email or as a guest. Has a cart and a mock checkout.

## Also doing

Encoding survey data for a municipal registry (RBIM). Over 500 booklets so far, checking each one against the physical form to keep errors near zero. It's not exciting work but it taught me a lot about handling data carefully.

## Stack

JS/TS, Python, React, Django, PostgreSQL, Firebase, WebSockets, REST APIs. I've used GitHub Actions for CI/CD on more than one project now, not just as a one-time tutorial thing.
